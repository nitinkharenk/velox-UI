Velox UI — Developer Reference

AI-powered animated React component generation platform. Operators capture ideas → enrich with AI → generate sandbox-safe code → review in live preview → publish to component library.

Pipeline Architecture

4-phase assembly line per component: Enrich → Generate → Validate → Repair (up to 3 retries).

Phases

PhaseFileWhat happens
1. Enrichlib/pipeline/generate.ts:enrichIdea()Convert vague prompt → structured EnrichedSpec JSON
2. Generatelib/pipeline/generate.ts:generateCode()Build component in 3 buffering phases: Structure → Logic & Hooks → Polish
3. ValidatevalidateCodeString()100+ static rules. AI auditor scores code — fails if < 75
4. RepairvalidateAndFix()Auto-fix + AI repair loop, up to 3 attempts. Falls back to repair_required.

Token Budget

40,000 tokens project-wide. Enforced by lib/pipeline/validationRuntime.ts. Vertex AI pipeline gets the full budget.

Pipeline Config Fallback

If pipelines table has no is_default=true row, generate route falls back to hardcoded Claude 3.5 Sonnet.

Idea State Machine

pending → enriching → enriched → generating → generated → validating → validated
                                                                           ↓
                                                         repair_required    (FAIL)
                                                         ready_with_warnings (PASS_WITH_WARNINGS)
                                                         ready              (PASS)
                                                            ↓
                                                         reviewing
                                                            ↓         ↓
                                                         approved   rejected
                                                         failed  (unhandled exception)

Sandbox Rules

Generated components run inside an iframe with Babel Standalone + UMD builds. No module system — any import crashes Babel.

Never Do This

  • ❌
    No import statements — Babel Standalone has no module resolver
  • ❌
    No h-screen, min-h-screen, 100vh, 100dvh — use style={{ minHeight: '100vh' }} instead
  • ❌
    No window.addEventListener / document.querySelector — use useRef + React synthetic events
  • ❌
    No .onChange(cb) on MotionValues — FM v11+ removed this; use .on('change', cb)
  • ❌
    No template-literal classNames: className={`foo ${var}`} — breaks Tailwind purge
  • ❌
    No arbitrary Tailwind brackets: h-[400px] — stripped by purge
  • ❌
    No placeholder icon classes (icon1, icon2) — use inline SVG elements

Always Do This

  • ✅
    Access Framer Motion: const {"{ motion, useMotionValue }"} = window.Motion
  • ✅
    Access GSAP: const gsap = window.GSAP
  • ✅
    React is global — no import needed
  • ✅
    Full-height: style={{ minHeight: '100vh' }}
  • ✅
    Export: export default function ComponentName() {"{ ... }"}
  • ✅
    Icons: inline <svg> elements only

Architecture Facts

Non-obvious facts that cause bugs when missed. Read before touching pipeline, sandbox, auth, or DB queries.

Fact 1 — Auth
Auth is not enforced. middleware.ts refreshes Supabase sessions but never redirects. Dashboard is publicly accessible by design. Add route protection only if explicitly asked.
Fact 2 — Globals Only
Sandbox uses globals, not imports. window.Motion, global React, window.GSAP. Any import statement crashes Babel Standalone.
Fact 3 — ideas vs assets
Separate tables, separate lifecycles. ideas = pipeline work queue. assets = published inventory. ingestAsset() is the one-way bridge.
Fact 4 — getPublishedAssets()
Has no is_published filter. Returns all assets. Use getLandingAssets() for public-facing pages.
Fact 5 — Slug Upsert
Slug is the upsert key. Publishing the same component name twice silently overwrites the existing asset.
Fact 6 — JS Aggregation
getDashboardData() aggregates in JavaScript, not SQL. Fetches all idea rows and counts in-memory. Fine at current scale; avoid at thousands of rows.
Fact 7 — FM v11 Polyfill
Framer Motion v11 .onChange() is polyfilled. Both the sandbox HTML and autoFixCode() handle this. Keep both polyfills.
Fact 8 — sessionRef
sessionRef in GeneratePage is intentional. Maintains both useState (re-renders) and useRef (async fetch loop access). Do not simplify to one.
Fact 9 — Pipeline Fallback
Fallback to Claude 3.5 Sonnet if pipelines table has no is_default=true row. System works without DB pipeline config.
Fact 10 — export default stripped
buildSandboxHTML() strips export default. Replaces export default function Foo with function Foo so Babel Standalone can hoist it. Name extracted separately for render call.
Fact 11 — Token Budget
40k token budget project-wide. Enforced by lib/pipeline/validationRuntime.ts. Vertex AI pipeline gets the full budget.
